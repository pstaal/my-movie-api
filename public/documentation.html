Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user /movies/:Title GET none a JSON object holding information about a particular movie
{
Title: 'Midsommar',
Description: 'A couple travels to Scandinavia to visit a rural hometown\'s fabled Swedish mid-summer festival. What begins as an idyllic retreat quickly devolves into an increasingly violent and bizarre competition at the hands of a pagan cult.',
Genre: 'horror',

}
Return data about a genre (description) by name/title (e.g., “Thriller”) /movies/genre/:Genre GET none a JSON object holding all the movies of a particular genre
{
Name: 'horror',
Desciption: "Scary stuff"
}
Return data about a director (bio, birth year, death year) by name /movies/director/:Director GET none a JSON object holding all the personal information of a director
{
Name: 'Martin Scorsese',
Bio: 'born somewhere...',
Birth: 1939,
Death: null,
}
Allow new users to register /users POST a JSON object holding Username, Password, Email and Birthday
{ Username: 'peter staal,
Password: 'my password'
Email: 'peter@peter.com'
Birthday: 'Date'}		 A text message showing the confirmation of registration
Allow users to update their user info (username) /users/:Username PUT a JSON object holding Username, Password, Email and Birthday
{ Username: 'peter staal,
Password: 'my password'
Email: 'peter@peter.com'
Birthday: 'Date'}		 A text message confirming the update was succesful
Allow users to add a movie to their list of favorites (showing only a text that a movie has been added—more on this later) /users/:Username/movies/:MovieID POST A JSON object holding name, password and favorite
{ name: 'peter staal,
password: 'my password',
favorite: 'Casino' } 		A text message showing the movie that was added to favorites
Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed—more on this later) /users/:Username/movies/:MovieID DELETE None A text message showing the movie that was deleted
Allow existing users to deregister (showing only a text that a user email has been removed—more on this later) /users/:Username DELETE None A text message showing the user that was removed
